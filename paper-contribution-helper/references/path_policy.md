# Path Policy

This package must remain portable across machines and workspaces.

## Allowed Paths

- Package-relative paths in documentation and manifests, such as `references/module_index.md`.
- Runtime roots derived from `Path(__file__)`.
- Current workspace paths derived from `Path.cwd()`.
- Explicit user-provided paths passed through CLI arguments.
- External API URLs and model API base URLs when they are configurable defaults.
- Environment-configured executable paths for ACP/Node/acpx, such as `PAPER_HELPER_ACPX_BIN`, `PAPER_HELPER_ACPX_NODE`, or `PAPER_HELPER_NODE_DIR`.

## Disallowed Paths

- Hardcoded machine-specific absolute paths, including Windows drive paths and `/Users`, `/home`, `/mnt`, or temporary build roots.
- Persisting user local PDF paths into packaged skill resources.
- Shipping project-run folders, raw PDFs, raw forum dumps, caches, or generated bytecode.
- Assuming a fixed installation directory for Codex or any skill.
- Hardcoding a local Node, npx, acpx, or Codex executable path discovered during one user's run.

## Manifest Rules

- Packaged manifests use paths relative to the generated skill root.
- Project-run manifests may reference user-provided inputs during the current run, but those paths must not be copied into reusable package references.
- Anonymous synthesis files must not include titles, author names, forum IDs, raw URLs, or unique benchmark bundles from old papers.

## Validation

Run:

```bash
python scripts/validate_modularity.py --skill-dir <this-skill-dir>
```

The validator checks for required modular references, top-level router size, forbidden path patterns, old package names, and cache artifacts.
